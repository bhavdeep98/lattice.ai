Quick Start

Get up and running with Lattice AWS CDK in under 5 minutes. This guide will walk you through creating your first production-ready AWS infrastructure.

1

Install Lattice

Add Lattice to your project using npm or yarn:

bash 
npm install lattice-aws-cdk aws-cdk-lib constructs
2

Create Your First Stack

Create a simple stack with built-in security and monitoring:

typescript 
import { Stack, StackProps } from 'aws-cdk-lib';
import { Construct } from 'constructs';
import { 
  LatticeNetwork, 
  LatticeBucket, 
  LatticeDatabase,
  applyLatticeAspects 
} from 'lattice-aws-cdk';

export class MyFirstStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    // Apply guardrails FIRST - The "Final Inspector"
    applyLatticeAspects(this, {
      environment: 'prod',
      projectName: 'MyApp',
      owner: 'DevTeam'
    });

    // AI-friendly JSON intents
    const network = new LatticeNetwork(this, 'Network', {
      cidr: '10.0.0.0/16',
      environment: 'prod',
      highAvailability: true
    });

    const storage = new LatticeBucket(this, 'Storage', {
      name: 'my-app-data',
      environment: 'prod',
      encryption: true,
      versioning: true
    });

    const database = new LatticeDatabase(this, 'Database', {
      name: 'my-app-db',
      environment: 'prod',
      engine: 'postgres',
      size: 'large'
    });

    // ‚úÖ Automatically secured, monitored, and backed up!
  }
}

Network Module

The Network Module creates a complete VPC infrastructure with subnets, NAT gateways, and security groups following AWS best practices.

Basic Usage

typescript 
import { LatticeNetwork } from 'lattice-aws-cdk';

const network = new LatticeNetwork(this, 'Network', {
  cidr: '10.0.0.0/16',
  environment: 'prod',
  highAvailability: true,
  natGateways: 3,
  enableVpnGateway: false
});

Configuration Options

Property Type Default Description
cidr string Required CIDR block for the VPC
environment string Required Environment (dev, staging, prod)
highAvailability boolean false Enable multi-AZ deployment
natGateways number 1 Number of NAT gateways
enableVpnGateway boolean false Enable VPN gateway

Escape Hatches

When you need advanced AWS features beyond Lattice's simple interfaces, escape hatches provide full access to underlying CDK constructs without losing the benefits of Lattice's guardrails.

The Pattern

Every Lattice construct exposes an instance property that gives you direct access to the underlying CDK construct:

typescript 
// Start with simple Lattice intent
const bucket = new LatticeBucket(this, 'DataBucket', {
  name: 'analytics-data',
  environment: 'prod',
  encryption: true
});

// Escape hatch: Add advanced S3 features
bucket.instance.addCorsRule({
  allowedOrigins: ['https://admin.myapp.com'],
  allowedMethods: [s3.HttpMethods.PUT, s3.HttpMethods.DELETE],
  allowedHeaders: ['x-amz-server-side-encryption'],
  maxAge: 3600
});

bucket.instance.addLifecycleRule({
  id: 'IntelligentTiering',
  transitions: [{
    storageClass: s3.StorageClass.INTELLIGENT_TIERING,
    transitionAfter: Duration.days(1)
  }]
});

Available Escape Hatches

üåê LatticeNetwork

network.instance ‚Üí ec2.Vpc

Access VPC, subnets, security groups, and routing

üóÑÔ∏è LatticeBucket

bucket.instance ‚Üí s3.Bucket

Add CORS, lifecycle rules, notifications

üóÉÔ∏è LatticeDatabase

database.instance ‚Üí rds.DatabaseInstance

Configure read replicas, parameter groups

‚öôÔ∏è LatticeCompute

compute.instance ‚Üí lambda.Function

Add layers, environment variables, triggers

Examples

Real-world examples showing how to build common architectures with Lattice.

Simple API

A basic REST API with database and monitoring:

typescript 
import { Stack, StackProps } from 'aws-cdk-lib';
import { Construct } from 'constructs';
import { 
  LatticeNetwork, 
  LatticeBucket, 
  LatticeDatabase,
  LatticeCompute,
  applyLatticeAspects 
} from 'lattice-aws-cdk';

export class SimpleApiStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    // Apply guardrails first
    applyLatticeAspects(this, {
      environment: 'prod',
      projectName: 'SimpleAPI',
      owner: 'DevTeam',
      threatModel: { enabled: true }
    });

    // Network infrastructure
    const network = new LatticeNetwork(this, 'Network', {
      cidr: '10.0.0.0/16',
      environment: 'prod',
      highAvailability: true
    });

    // Database
    const database = new LatticeDatabase(this, 'Database', {
      name: 'api-db',
      environment: 'prod',
      engine: 'postgres',
      size: 'medium'
    });

    // API Lambda function
    const api = new LatticeCompute(this, 'API', {
      name: 'api-handler',
      environment: 'prod',
      runtime: 'nodejs18',
      memorySize: 512,
      timeout: 30
    });

    // File storage
    const storage = new LatticeBucket(this, 'Storage', {
      name: 'api-uploads',
      environment: 'prod',
      encryption: true,
      versioning: true
    });

    // Result: Production-ready API with monitoring, security, and backups!
  }
}

Troubleshooting

Common issues and solutions when working with Lattice AWS CDK.

Common Issues

‚ùå CDK Bootstrap Required

Error: This stack uses assets, so the toolkit stack must be deployed

Solution:

Run CDK bootstrap in your target AWS account and region:

cdk bootstrap aws://ACCOUNT-NUMBER/REGION

‚ùå Dashboard Name Conflicts

Error: Dashboard with name already exists

Solution:

Lattice automatically handles naming conflicts by adding timestamps. If you see this error, try:

// Add unique suffix to your stack
export class MyStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id + '-' + Date.now(), props);
    // ... rest of your stack
  }
}

‚ùå SNS Topic Naming Conflicts

Error: Topic with name already exists

Solution:

Use unique project names in your Lattice aspects:

applyLatticeAspects(this, {
  environment: 'prod',
  projectName: 'MyApp-' + this.account, // Make it unique
  owner: 'DevTeam'
});

AI Intents

Learn how Lattice transforms simple JSON intents into complex AWS infrastructure.

What are AI Intents?

AI Intents are simple, declarative JSON objects that describe what you want to build, not how to build it. They're designed to be easily generated by AI systems while remaining human-readable and maintainable.

‚ùå Traditional CDK

const vpc = new ec2.Vpc(this, 'VPC', {
  cidr: '10.0.0.0/16',
  maxAzs: 3,
  subnetConfiguration: [
    {
      cidrMask: 24,
      name: 'Public',
      subnetType: ec2.SubnetType.PUBLIC,
    },
    {
      cidrMask: 24,
      name: 'Private',
      subnetType: ec2.SubnetType.PRIVATE_WITH_EGRESS,
    },
    {
      cidrMask: 28,
      name: 'Isolated',
      subnetType: ec2.SubnetType.PRIVATE_ISOLATED,
    }
  ],
  natGateways: 3,
  enableDnsHostnames: true,
  enableDnsSupport: true
});

// Add security groups, NACLs, route tables...
// 50+ more lines of configuration

‚úÖ Lattice Intent

const networkIntent = {
  cidr: '10.0.0.0/16',
  environment: 'prod',
  highAvailability: true
};

const network = new LatticeNetwork(
  this, 
  'Network', 
  networkIntent
